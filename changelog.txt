changelog — nsight_agent
========================

2026-04-07
----------
- Add `compare` subcommand: single-shot LLM comparison of two `.sqlite` profiles
  producing a structured `ComparisonReport` (narrative + key differences table
  ordered by magnitude); no tool-use loop — both summaries and a pre-computed
  structural diff are injected into one prompt
- Add `nsight_agent/analysis/diff.py`: `compute_profile_diff()` aligns two
  `ProfileSummary` objects and determines comparison mode:
    - `phase_aware` — same phase count + names; full per-phase analysis
    - `summary` — phases differ but Jaccard kernel overlap >= 20%; per-kernel diff included
    - `summary_no_kernel` — phases differ and overlap < 20%; top-level metrics only
  Jaccard similarity used for kernel overlap; status message shown before LLM call
  (e.g. "Phases do not match (A has 4, B has 6) and kernel overlap is 8%...")
- Add `nsight_agent/agent/compare.py`: `run_compare()` single-shot backends for
  Anthropic, OpenAI, Gemini, and claude_code; saves `{stem_a}_vs_{stem_b}_{ts}_prompt.txt`
  and `_response.txt` alongside the profile
- Add `ScalarDiff`, `KernelDiff`, `MemcpyDiff`, `MpiDiff`, `ProfileDiff`,
  `ComparisonDiff`, `ComparisonReport` Pydantic models to `analysis/models.py`
- Add `_print_phase_table()` helper to `__main__.py`; `compare` always prints both
  phase tables before the LLM call; key differences table color-codes by magnitude
- Output saved to `{stem_a}_vs_{stem_b}_{ts}_output.txt` next to profile A

- Save terminal output to `{stem}_{ts}_output.txt` alongside prompt/response files
  after each `analyze` run: `Console(record=True)` captures all Rich table output;
  a `log` callable is threaded through `run_agent` and all four backends so the
  turn-by-turn agent output (previously raw `print()`) is routed through the same
  console and captured; output file is not written in `--json` mode

- Remove `_ensure_indexes` and the writable connection in `NsysProfile`: the tool
  previously opened a writable connection on first use to add query-accelerating
  indexes to the profile file; SQLite does not support temporary indexes on
  non-temporary tables, so there is no non-modifying alternative; the ~25% local
  analysis speedup was not worth the hidden file modification given that runtime
  is dominated by LLM round-trips
- Add `has_table` guards to `compute_gpu_sync_time` and `compute_gpu_memcpy_time`:
  previously both would crash with `OperationalError` on profiles that lack
  `CUPTI_ACTIVITY_KIND_SYNCHRONIZATION` or `CUPTI_ACTIVITY_KIND_MEMCPY`
- Add docstring note to `PhaseSummary` clarifying that `total_gpu_idle_s` counts
  only inter-kernel gaps, not leading/trailing idle at phase boundaries
- Add "Provenance" section to README noting the tool was vibe coded and that
  metrics should be treated as starting points rather than ground truth

2026-04-06
----------
- Add phase-scoped time window support to agent tools: `top_kernels`, `gap_histogram`,
  `memcpy_summary`, `mpi_summary`, `nvtx_ranges`, and `stream_summary` now accept optional
  `start_ns`/`end_ns` parameters; when provided they query only within that window using
  existing internal `_window_*` helpers; tool schemas updated with parameter descriptions
  that direct the agent to use phase timestamps from the pre-seeded context
- Add `_window_memcpy_by_kind`, `_window_streams`, `_window_nvtx_ranges` to `metrics.py`
  (windowed counterparts for the three functions that previously lacked them)
- Add `start_ns`/`end_ns` fields to `PhaseSummary` (absolute CUPTI nanosecond timestamps);
  the pre-seeded phase summary previously only had `start_s`/`end_s` (seconds), which the
  agent could not use directly as tool window arguments
- Add `--allow-app-knowledge` flag to `analyze`: by default the system prompt
  now constrains the model to ground all suggestions in profile data only;
  `--allow-app-knowledge` restores the previous behavior where the model may
  draw on training knowledge about the application (e.g. suggest named env vars
  inferred from kernel names)
- Full kernel name resolution (todo item 2): switch GROUP BY from shortName to
  demangledName so each template instantiation is a distinct row; add
  `_normalize_demangled()` to strip SFINAE prefix and `(T2)` suffix; add
  `short_name` field to `KernelSummary` for CLI display; fall back to shortName
  on older profiles that lack the demangledName column
- Add `action_category` field to hypothesis schema (todo item 5): enum of
  `runtime_config` / `launch_config` / `code_optimization` / `algorithm`,
  ordered by effort; boundary rule (what vs. how) included in system prompt
- Add "Action" column (magenta, abbreviated) to hypotheses table in `cmd_analyze`
- Update `README.md`: document `action_category` field in hypothesis output
- Add `eval/score.py` — standalone independent hypothesis evaluator; scores
  nsight-agent hypothesis JSON using gpt-5.4 on six criteria (bottleneck
  alignment, addressability, mechanistic plausibility, specificity/actionability,
  testability, generality) plus implementation-cost and correctness-risk flags
  and a list-level redundancy score; writes `.eval.json` alongside input
- Add `eval/context_example.yaml` — template for injecting application/phase
  context into the evaluator prompt (critical for addressability scoring)
- Add `eval/` to `.gitignore` — evaluator is intentionally separate from nsight-agent
- Add rich terminal table to `eval/score.py` — displays per-hypothesis scores (color-coded
  1–5), implementation-cost and correctness-risk flags, and per-hypothesis average; footer
  shows redundancy score and overall aggregate
- Update `eval/score.py` table and output files: rename column "Hypothesis" → "Suggestion"
  (shows verbatim suggestion text truncated to 50 chars); save full LLM prompt to
  *.eval.prompt.txt; save plain-text table + rationales to *.eval.table.txt
- Add verdict + knowledge_source to `eval/score.py` — LLM now gives an explicit
  strongly_recommended/recommended/uncertain/not_recommended verdict per hypothesis with
  a 2-3 sentence rationale; knowledge_source field (context/model/both) flags when the
  verdict draws on the model's own training knowledge of the application vs. injected context;
  verdict column added to the rich table with rationales printed below; prompt strengthened
  to be opinionated and treat injected context as authoritative

2026-04-02
----------
- Add token reporting: track and display prompt/completion token counts
  after each agent run (4748cfc)
- Add README.md (fdb639e)
- Multi-provider LLM support (OpenAI, Gemini) (73ed731)
  - `_parse_provider_and_model()` resolves backend from model prefix, explicit
    --provider flag, or env-var auto-detection (ANTHROPIC → OPENAI → GOOGLE)
  - `_run_openai()` — function-calling loop via openai>=1.0
  - `_run_gemini()` — Gemini start_chat() loop via google-genai
  - `--provider {anthropic,openai,gemini}` flag added to CLI
  - New optional extras: `pip install 'nsight-agent[openai]'` / `[gemini]`
- CPU-GPU overlap metrics (4d6c67f)
  - `KernelSummary`: avg/max launch overhead (µs) per kernel name
  - `ProfileSummary`: `cpu_sync_blocked_s` / `cpu_sync_blocked_pct`
  - Sample (A100 CG run): 20.5% of GPU kernel time spent in cuEventSynchronize
- Per-phase gap histogram (f88088d)
  - `PhaseSummary.gap_histogram` now scoped to phase window (6 buckets:
    <10µs through >100ms); previously only a global histogram existed
- Make PRAGMA cache_size and mmap_size adaptive based on profile file size (4676d36)
- Richer GPU metrics (6329ff0)
  - `KernelSummary`: std_dev_ms, cv, avg_registers_per_thread,
    avg_shared_mem_bytes, estimated_occupancy
  - `MemcpySummary`: pct_of_peak_bandwidth
  - `ProfileSummary`: peak_memory_bandwidth_GBs (from TARGET_INFO_GPU)
- Runtime performance improvements — analysis time reduced ~25% (200b108)
  - Index creation on first open for kernel start and MPI tables
  - Single-pass MPI stats with CASE-based conditional aggregation
  - Self-profiling timers printed in timing breakdown table
